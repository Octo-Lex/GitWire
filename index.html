Press Statement

Short Bio

GitWire is an open-source, self-hosted GitHub App powered by Claude. It automates issue triage, self-heals CI failures, enforces branch policy, manages stale issues, and ships autonomous code fixes — all from your own infrastructure.

Comprehensive Background

Long Bio

GitWire runs as a GitHub App you install on your repositories. Behind the scenes, a Node.js server receives webhooks, queues jobs through BullMQ + Redis, and calls Claude to classify issues, diagnose CI failures, generate code patches, and review pull requests — all with a full audit trail.

Built around eight pillars — Issue & PR Triage, Self-Healing CI, Autonomous Contributor, Maintainer Tools, Multi-Repo Insights, Branch Enforcement, Merge Queue, and AI Review Gate — GitWire ships as a single Docker Compose deployment with a real-time dashboard, 36 database tables, 9 background workers, and 100+ API endpoints. Self-hosted. Private. No data leaves your VPC.

The Core Platform Architecture

The Eight Pillars of GitWire

Deeply integrated, granular control pipelines. Every pillar is built directly into GitWire's single deployment engine for unified orchestration.

1. Issue & PR Triage

Claude classifies incoming issues and pull requests by type, severity, and priority. Auto-labels are applied instantly. Duplicate detection uses trigram-hash embeddings with cosine similarity.

2. Self-Healing CI

Detects failed GitHub Actions workflow runs via webhooks. Claude diagnoses errors, generates code patches, and opens correction PRs with full file replacement — automatically.

3. Autonomous Contributor

A guarded two-pass AI pipeline: Pass 1 analyzes the issue and ranks relevant files, Pass 2 generates full-file fixes. Scope-limited to labeled issues, with rate limits and pre-merge validation.

4. Maintainer Tools

Stale issue/PR scanner with warning and auto-close. Orphaned branch cleanup. Context-aware /gitwire comment commands for maintainers. Full repository settings API.

5. Multi-Repo Insights

Real-time dashboard with SWR polling across all synced repos and installations. Tracks issues, PRs, CI runs, fix attempts, duplicate signals, and governance state — with Recharts visualizations.

6. Branch Enforcement

Programmable branch governance with enforcement policies, violation detection, and reconciliation runs. Config validation checks repo settings against declared policy.

7. Merge Queue

Configurable merge queue with batch grouping, concurrency limits, and status checks. Includes error recovery with automatic retry and rollback event tracking.

8. AI Review Gate + Audit Trail

Pre-merge AI code review via GitHub Check Runs. Detects hardcoded secrets, missing error handling, and antipatterns. Chain-linked audit trail with SHA-256 hashes for SOC2 compliance.
